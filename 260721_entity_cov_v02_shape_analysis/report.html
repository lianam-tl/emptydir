Conclusion. The disagreement is real at the segmented-film level, but it is not a clean measurement of video duration. Source balancing, label deduplication, and removing parse errors do not recover similar model rankings. The strongest remaining confounder is that full and half use very different visual-token density, so segmentation changes both context duration and apparent spatial detail.
Failure finding. All 14 fine-tuned runs fail to parse film-04 full; Pegasus-15 is the only run that parses it. That row contributes 8 / 43 (18.6%) of the full character observations, so zero-penalization creates a large absolute penalty. It does not explain the ranking disagreement: parse-clean appearance rank correlation falls from 0.179 to 0.004.

7. Recommended reporting

  1. Keep full and half as separate stress tests; do not interpret either as a drop-in proxy for the other.
  2. Add a source-balanced metric: combine both halves back into one source, average repeated character labels, then weight each of the seven sources equally.
  3. Report both the current penalized score and a parse-clean diagnostic. The penalized score remains useful, but it currently measures JSON completion as well as entity quality.
  4. Show length-finish rate, parse-error rate, output tokens, and input tokens per frame next to quality metrics.
  5. Trace the video processor's frame-resolution/token-budget rule. Full and half currently change both temporal duration and apparent spatial token density, so this is not a clean duration-only experiment.
  6. Either reduce the required shot-level output verbosity or score a compact entity-only schema. Raising the output cap above 64K would be expensive and does not address unequal weighting.
